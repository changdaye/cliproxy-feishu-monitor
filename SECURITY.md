# Security Policy

## 支持范围

当前安全修复优先覆盖最近的公开 release 版本线。

## 报告方式

如果你发现了安全问题，请不要直接公开提交 issue 暴露细节。

建议通过以下方式私下联系维护者：
- GitHub 私信 / 邮件（如果仓库资料页提供）
- 或先提交不含利用细节的简短 issue，说明需要私下沟通

## 公开仓库安全约定

请不要在公开 issue、PR、文档、commit 中泄露：
- 真实管理密钥
- 飞书 webhook / secret
- 真实运行配置文件
- 真实服务器 IP（除非明确作为公开服务地址）
- 运行状态文件内容
